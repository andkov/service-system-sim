## Table: `ds_payment` (payment.sqlite)

**Pattern**: Sim (generated by `00-sim.R`)
**Grain**: One row per person × month × need_code (atomic payment)

| Column | Type | Description |
|---|---|---|
| `payment_id` | integer | Surrogate key |
| `person_id` | integer | Person identifier (negative = canonical demo) |
| `pay_period` | date | Month of payment (YYYY-MM-01) |
| `client_type` | character | `ETW` or `BFE` |
| `household_role` | character | `HH` or `SP` |
| `need_code` | character | Payment category (client_type-specific) |
| `payment_amount` | numeric | Dollar amount |

**Invariants**: One `client_type` per person-month. One `household_role` per person-month. Multiple rows per person-month allowed (one per `need_code`).

## Table: `ds_episode` (episode.sqlite)

**Pattern**: Derive (generated by `2a-episode.R`)
**Grain**: One row per SPELL_BIT (episode)

**Operationalization**: Episode = SPELL_BIT. Terminated by: gap ≥ 2 months, OR change in `client_type`, OR change in `household_role`. See `2a-episode.R` header for full disclosure.

| Column | Type | Aggregation Property |
|---|---|---|
| `person_id` | integer | — |
| `spell_id` | integer | Groups SPELL_BITs into SPELLs |
| `episode_id` | integer | Unique per person |
| `episode_start` | date | **Extremal** (MIN to SPELL) |
| `episode_end` | date | **Extremal** (MAX to SPELL), NA if ongoing |
| `client_type` | character | **Not aggregatable** (varies across SPELL_BITs) |
| `household_role` | character | **Not aggregatable** |
| `episode_length_months` | integer | **Non-additive** ⚠️ (recompute for SPELL) |
| `n_months` | integer | **Additive** ✅ |
| `total_payment` | numeric | **Additive** ✅ |
| `n_need_codes_total` | integer | **Additive** ✅ |

## Table: `ds_event` (episode.sqlite)

**Pattern**: Derive (generated by `3a-event.R`)
**Grain**: One row per event (episode boundary)

| Column | Type | Description |
|---|---|---|
| `person_id` | integer | Person identifier |
| `event_month` | date | Month of the event |
| `event_type` | character | `NEW`, `RETURNED`, or `CLOSED` |
| `client_type` | character | At time of event |
| `household_role` | character | At time of event |
| `episode_id` | integer | Links to ds_episode |

**Event types**: `NEW` = first-ever episode start. `RETURNED` = subsequent episode start. `CLOSED` = episode end.

## Table: `ds_caseload_event` (timeseries.sqlite)

**Pattern**: Reconcile (generated by `05-caseload-event.R`)
**Grain**: One row per month × client_type × household_role

| Column | Type | Description |
|---|---|---|
| `pay_period` | date | Month |
| `client_type` | character | Stratification |
| `household_role` | character | Stratification |
| `active_clients` | integer | From ds_caseload |
| `NEW` | integer | From ds_event_count |
| `RETURNED` | integer | From ds_event_count |
| `CLOSED` | integer | From ds_event_count |
| `delta_caseload` | integer | NEW + RETURNED - CLOSED |
| `implied_next_caseload` | integer | active_clients + delta_caseload |

**Stock-flow identity**: `Caseload(t) = Caseload(t-1) + NEW(t) + RETURNED(t) - CLOSED(t)` must hold exactly.
